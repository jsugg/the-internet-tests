# The Internet Tests

Curated multi-stack automation playbook for [The Internet](https://the-internet.herokuapp.com/), using one shared scenario catalog to compare Selenium, Playwright, browser matrices, reports, and CI tradeoffs.

## 1. What this repo is

- A teaching repo for UI and resource-layer test automation.
- A side-by-side playbook for Java Selenium/TestNG and TypeScript Playwright.
- A catalog-driven suite: scenario IDs live in [`scenarios/catalog.yml`](scenarios/catalog.yml), tests reference those IDs, and [`tools/check-scenarios.py`](tools/check-scenarios.py) keeps coverage honest.
- A CI reference for fast PR gates, per-stack full gates, scheduled nightly coverage, and report artifacts.
- MIT-licensed; see [`LICENSE`](LICENSE).

## 2. What this repo is not

- Not a production application; the app under test is the public The Internet demo app.
- Not fake REST CRUD; HTTP/resource checks focus on pages the demo app actually exposes.
- Not every framework. The maintained stacks are Java Selenium/TestNG and TypeScript Playwright; Python Playwright is planned.
- Not a place for unbounded framework experiments. Add scenarios and stacks through the catalog and roadmap.

## 3. Fast start

Run the TypeScript Playwright smoke suite first; it is the quickest local feedback loop.

```bash
docker compose -f docker/compose.yml up -d website

cd stacks/ts-playwright
npm ci
npx playwright install chromium
THE_INTERNET_BASE_URL=http://localhost:7080 npm run test:chromium:smoke

cd ../..
docker compose -f docker/compose.yml down
```

Then use the stack READMEs for deeper runs:

- [Java Selenium/TestNG stack](stacks/java-selenium-testng/README.md)
- [TypeScript Playwright stack](stacks/ts-playwright/README.md)

## 6. Learning paths

| Path | Start here | Then study |
| --- | --- | --- |
| Beginner | Login, checkboxes, dropdown, add/remove | JavaScript alerts, uploads, dynamic loading |
| Interview candidate | Login, dynamic loading, dynamic controls | iframe, windows, upload/download, broken images, status codes |
| Legacy Selenium maintainer | Java driver lifecycle and waits | Page Objects, Grid, parallelism, Surefire reports |
| Modern Playwright learner | Locators and assertions | Fixtures, projects, traces, browser channels, mobile emulation |
| Senior CI/architecture reviewer | Scenario catalog and stack matrix | PR gates, path filters, nightly matrix, artifact strategy |
